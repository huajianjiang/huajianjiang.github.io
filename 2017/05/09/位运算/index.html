以 Android 中的 ExpandableListView 为例：

  • 代表 一个 group 类型的打包后的位置信息,该位存储在最高位 64
  • 代表一个为 child 类型的打包后的位置信息,该位存储在最高位 64
  • 代表一个无法确认的(非法的) type 的打包后的位置信息,该信息既不代表 null 也不代表没有设置
  • 该数值代表 Group/child 打包后的位置信息是无法确认的(非法的)，这个值是不可能存在的，除非在打包 Group 位置信息的时候存储了 child 位置信息
  • 用于解包 child 位置信息时所使用的 child 掩码，child 的位置信息存储在低 32 位
  • 用于解包 group 位置信息时所使用的 group 掩码，group 的位置信息存储在高 31 位
  • 用于解包 packagedPosition 中的 type 位的 type 掩码
  • 用于解包 packagedPosition 中的 group 位的 group 掩码，可以得出 group 的原始位置信息
  • 用于在打包位置信息的时候打包 type 位信息，type 位存储在最高位，这个是移位的个数
  • 用于在打包 child 位置信息时的掩码，child 位置信息存储在低 32 位
  • 用于在打包 group 位置信息时的掩码，group 位置信息存储在高 31 位

解包过程,以上例为例


* 解包 type 信息
/**
 * type mask = 0x8000000000000000L
 * 上面的打包后的 十六进制 long 型的 packagedPosition =  0x8000000600000003L
* 这里 type mask & packagedPosition 也就是 0x8000000000000000 & 
* 0x8000000600000003 解包出最高位 type 信息，结果是 1，所以是 child 类型
* 其实 这个 & 位操作就是检查是否 type 位是否是 child 类型，之所以定义 type 
* mask 是 0x8000000000000000L ，是为了取最高位 type 信息，其他的位舍弃
*
*/
public static int getPackedPositionType(long packedPosition) {
        if (packedPosition == PACKED_POSITION_VALUE_NULL) {
            return PACKED_POSITION_TYPE_NULL;
        }

        return (packedPosition & PACKED_POSITION_MASK_TYPE) == PACKED_POSITION_MASK_TYPE
                ? PACKED_POSITION_TYPE_CHILD
                : PACKED_POSITION_TYPE_GROUP;
}

* 解包 `group position` 信息
/**
 * 这里是解包 group position 位得到 group position 
 * 其实是对打包的 packedPosition 进行解码 ，利用 MASK_GROUP 掩码来 & 
 * packedPosition 得到纯粹的 group position 位信息，然后再向右移动 32 位到低 
 * 32 位得到最终的 group position
 *  packagedPosition =  0x8000000600000003L
 */
public static int getPackedPositionGroup(long packedPosition) {
    // Null
    if (packedPosition == PACKED_POSITION_VALUE_NULL) return -1;

        return (int) ((packedPosition & PACKED_POSITION_MASK_GROUP) >> PACKED_POSITION_SHIFT_GROUP);
}

* 解包 `child position` 信息
    /**
     * 这里先事先 获取 type 信息判断是否是 child 类型，如果是的话再 解码 
     * packedPosition 编码里的 child position 信息
     */
    public static int getPackedPositionChild(long packedPosition) {
        // Null
        if (packedPosition == PACKED_POSITION_VALUE_NULL) return -1;

        // Group since a group type clears this bit
        if ((packedPosition & PACKED_POSITION_MASK_TYPE) != PACKED_POSITION_MASK_TYPE) return -1;

        return (int) (packedPosition & PACKED_POSITION_MASK_CHILD);
    }

* 如何编码 `group` 类型的信息的

       public static long getPackedPositionForGroup(int groupPosition) {
        // No need to OR a type in because PACKED_POSITION_GROUP == 0
        // 这句话的意思就是: 没有必要再 添加 group type 位信息，因为 group type = 0
        return ((((long)groupPosition) & PACKED_POSITION_INT_MASK_GROUP)
                        << PACKED_POSITION_SHIFT_GROUP); 
    }
